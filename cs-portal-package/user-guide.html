Logging In

The CS Portal requires authentication to ensure only authorized agents can access client information and perform support actions.

Login Page Screenshot

The login page features secure authentication with username/email and password fields

How to Login

  1. Open the Portal
    Navigate to the CS Portal URL in your browser. You'll automatically be redirected to the login page if not already authenticated.
  2. Enter Your Credentials
    Enter your assigned username or email address and password in the respective fields.
  3. Optional: Remember Me
    Check the "Remember me" option to have your username pre-filled on future visits (password is never saved).
  4. Sign In
    Click the "Sign In" button or press Enter to authenticate and access the portal.
Security Features

All login credentials are encrypted during transmission. Sessions automatically expire after inactivity for your protection.

Forgot Password?

If you've forgotten your password, click the "Forgot password?" link and contact your system administrator for assistance with password reset.

Signing Out

To log out securely:

  1. Click your user avatar in the top-right corner
  2. Select "Sign Out" from the dropdown menu
  3. You'll be redirected to the login page
Important Security Reminder

Always sign out when leaving your workstation to protect client data and maintain security compliance.

Finding Clients

The CS Portal offers multiple search methods to quickly locate client information. Choose the method based on the information available.

Find Client Page Screenshot

Main dashboard showing all available search methods

Primary Method: Mobile Number (Recommended)

Why it's recommended:

  • Fastest: Returns results in under 1 second
  • Most Accurate: Mobile numbers are unique identifiers
  • Primary ID: Main registration method in the system

How to use:

  1. Click the "Mobile Number" card on the Find Client page
  2. Enter the client's registered mobile number (include country code if required)
  3. Click "Search" or press Enter
  4. View the client profile or select from results

Alternative Search Methods

1. Client Name

Search using full or partial names when mobile number is unavailable.

  • Enter first name, last name, or both
  • Partial matches will show multiple results
  • Results sorted by relevance
  • Use full names for more accurate results

Client Name Search Screenshot

Name search interface with results list

2. Client ID

Most precise method when you have the exact identifier.

  • Enter the complete Client ID number
  • Returns exact match only
  • Ideal when client provides their ID
  • No ambiguity in results

3. Email Address

Search using registered email address.

  • Must match exactly as registered
  • Case-insensitive search
  • Good for web-based inquiries

4. Salesforce Case

Import information from existing cases.

  • Enter Salesforce Case Number
  • Auto-populates client information
  • Links interaction to existing case
  • Saves time on repeat contacts
Pro Tip

Start with mobile number search whenever possible. If that fails, try Client ID, then name search as a last resort for broader results.

Understanding Search Results

After searching, results can be displayed in two views:

  • List View: Detailed table format with key information
  • Grid View: Card-based layout for easier scanning

Click any result to view the complete client profile and proceed to authentication.

Client Authentication

Client authentication is mandatory before accessing sensitive information. This ensures data protection compliance and prevents unauthorized access.

Why Authentication Matters

Authentication protects both clients and the company by verifying identity before discussing account details, ensuring compliance with data protection regulations, and preventing fraud.

Choose Authentication Method Screenshot

Authentication method selection screen

Method 1: Mobile App Authentication (Recommended)

Best for: Quick and secure verification

How it works:

  1. Select "Mobile App Authentication"
  2. System sends push notification to client's registered mobile app
  3. Client taps "Approve" in their app
  4. Instant verification (10-30 seconds typical)

Features:

  • ✅ Instant verification
  • ✅ No sensitive information sharing needed
  • ✅ Most secure method
  • ✅ Best client experience

Method 2: OTP Verification

Best for: Clients without app access or experiencing app issues

How it works:

  1. Select "OTP Verification"
  2. System sends 6-digit code via SMS to registered mobile
  3. Ask client to read the code
  4. Enter code in verification field
  5. Click "Verify" to authenticate

Features:

  • ⚡ Fast verification (1-2 minutes)
  • 📱 Works via standard SMS
  • 🔑 Code expires after 5 minutes
  • 🔄 Can request new code if needed

OTP Verification Screenshot

OTP input screen with verification field

Method 3: Personal Questions

Best for: When digital methods aren't available

How it works:

  1. Select "Personal Questions"
  2. System displays security questions from profile
  3. Ask client each question verbally
  4. Mark answers as correct/incorrect
  5. Minimum 3 correct answers required

Question types include:

  • Date of birth
  • Last transaction amount and date
  • Account opening date
  • Registered address details
  • Recent activity verification

After Successful Authentication

Authentication Successful

Once authenticated, the client's full profile becomes accessible, sensitive information is revealed, and all actions are logged with timestamps for security.

Authentication Failures

If authentication fails:

  • System limits the number of attempts (typically 3)
  • Try an alternative authentication method
  • Verify you're speaking with the correct person
  • Escalate to supervisor if repeated failures occur
  • All failed attempts are logged for security review

Skip Authentication (Use With Extreme Caution)

Warning: Skip Authentication

Skipping authentication should ONLY be used for public information inquiries, account opening assistance, or general product questions. This action is logged and audited.

When skipping is acceptable:

  • Public information requests only
  • New account opening guidance
  • General product/service questions
  • Technical support for app installation

Restrictions when authentication is skipped:

  • ❌ No access to account balances
  • ❌ No transaction history
  • ❌ Cannot make account changes
  • ❌ Cannot discuss personal details
  • ⚠️ All actions flagged for manager review

Best Practices & Tips

Security Best Practices

  • Always authenticate when possible - Don't skip unless absolutely necessary
  • Use app authentication first - It's fastest and most secure
  • Never share login credentials - Each agent has unique access
  • Lock your workstation - When stepping away, even briefly
  • Sign out properly - Use the logout button, don't just close the browser
  • Report suspicious activity - Immediately alert your supervisor

Client Communication

  • Explain the process - Tell clients why authentication is needed
  • Guide them patiently - Not everyone is tech-savvy
  • Offer alternatives - If one method fails, suggest another
  • Stay professional - Even if authentication takes multiple attempts
  • Confirm understanding - Make sure clients know what to expect

Efficiency Tips

  • Start with mobile search - It's the fastest method
  • Keep case notes - Document your interactions thoroughly
  • Use keyboard shortcuts - Navigate faster with Enter key
  • Bookmark frequently used features - Save time on common tasks
  • Learn all search methods - Be prepared for any situation

What to Do When...

Client doesn't have their phone:

  • Try personal questions authentication
  • Ask for alternative verification information
  • Verify callback number and schedule follow-up

App authentication times out:

  • Verify client has app installed and updated
  • Check if notifications are enabled
  • Switch to OTP method as backup

Multiple failed authentication attempts:

  • Escalate to supervisor immediately
  • Document all attempts and responses
  • Consider potential security risk
  • Do not proceed without proper authentication
