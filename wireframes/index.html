📋 About This Project

These wireframes represent the complete user flow for the TOP 2.0 CS Portal - a secure, authentication-first customer service portal designed for CS agents to access client information safely and efficiently.

The portal integrates with Salesforce Service Cloud and Nice CXOne, providing CS agents with the tools they need to assist clients effectively while maintaining strict security protocols.

🔒 Authentication-First
Client verification before data access
📱 3 Auth Methods
App, OTP, and personal verification
⏱️ Session Management
15-minute time-limited access
🎨 Design System
Complete component library
📊 Real-time Data
Live client information access
🔍 Advanced Search
Multiple search methods

Page 1

Find Client

Entry point where CS agents select their preferred search method. Features mobile number search as the primary recommended option.

Page 2

Mobile Number Search

Smart phone number input with auto-formatting, country code selection, and E.164 normalization. Supports multiple input formats.

View Wireframe
Page 3

Choose Authentication Method

Authentication method selection with three options: App Authorization (recommended), SMS/Email OTP, and Personal Information Verification.

View Wireframe
Page 4

Client Detail View

Complete client dashboard with tabbed navigation showing overview, transactions, wallets, cards, and activity logs. Includes session timer and quick actions.

Page 5

Client List View

Search results table showing multiple clients when search returns more than one match. Includes sorting, filtering, and quick actions.

📐 Key Design Principles

  • Typography: Inter font family (300-700 weights)
  • Spacing: 4px base unit system (space-1 through space-16)
  • Border Radius: sm(4px), md(8px), lg(12px), xl(16px)
  • Breakpoints: sm(640px), md(768px), lg(1024px), xl(1280px)
  • Layout: 12-column grid system with responsive breakpoints
